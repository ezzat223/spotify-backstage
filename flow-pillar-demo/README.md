# Flow Pillar Demo

This demo seeds the running Backstage instance with a small set of entities, TechDocs and runnable Scaffolder templates to demonstrate adoption of the Flow pillar.

Quick summary:
- Catalog entities: components (AI service, website), API, Resource (Terraform-managed infra), System, Group, User
- TechDocs: Markdown docs including an ADR attached to the AI service
- Scaffolder templates: three runnable templates (AI-only, software-only, AI+software) that publish a stub repo under the provided GitHub owner

Wiring:
- A minimal `app-config.local.yaml` was added at the repo root to register the demo catalog and templates with the running backend. The entries are commented and easy to remove.

Demo script (10–12 minutes):
1. Start dev environment: `./scripts/start-dev.sh` (this loads `.env` and ensures Postgres is running).
2. Open Backstage: http://localhost:3000 → Catalog → Search for "flow-pillar-demo".
3. Show the AI service entity → TechDocs tab → ADR and README.
4. Show System and Group relationships.
5. Go to Create (Scaffolder) → pick "Flow Pillar — AI-only" template and fill the form (project name). Run to create a repo under `ezzat223`.
6. Return to Catalog → search for the new catalog-info.yaml generated by the template (if template created and registered).

Clean-up:
- Remove the demo: delete the `flow-pillar-demo/` directory and remove the two lines added to `app-config.local.yaml` (or delete the file entirely).

Notes:
- This demo uses the existing Postgres-backed catalog and the repository's TechDocs local builder.
- Scaffolder will publish repos using the GitHub token configured in `.env` (GITHUB_TOKEN). Ensure it has `repo` scope and permission to create repos under `ezzat223`.
- Soundcheck / RBAC plugins are out of scope (paid). This README does not touch them.
